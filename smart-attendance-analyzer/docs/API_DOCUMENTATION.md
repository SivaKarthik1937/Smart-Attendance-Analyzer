# API Documentation

Base URL (local dev): `http://localhost:8000`

Interactive, always-current documentation is auto-generated by FastAPI at **`/docs`** (Swagger UI) and **`/redoc`** (ReDoc). This document is a human-readable reference alongside that.

All endpoints except `/`, `/health`, `/api/auth/register`, and `/api/auth/login` require a JWT bearer token:
```
Authorization: Bearer <access_token>
```

---

## Authentication — `/api/auth`

| Method | Path | Auth | Description |
|---|---|---|---|
| POST | `/api/auth/register` | None | Create a student or faculty account. Returns a JWT immediately (auto-login). |
| POST | `/api/auth/login` | None | OAuth2 password flow (form-encoded `username` + `password`). Returns a JWT. |
| GET | `/api/auth/me` | Any | Returns the current user's identity (username, email, role, linked profile code/name). |
| POST | `/api/auth/logout` | Any | Stateless acknowledgment (JWT has no server-side session to revoke). |

**Register request body** (`application/json`):
```json
{
  "username": "janedoe",
  "email": "jane@example.com",
  "password": "secret123",
  "role": "student",
  "name": "Jane Doe",
  "department": "Computer Science",
  "semester": 3,
  "gender": "Female",
  "distance_km": 8.5
}
```
For `role: "faculty"`, supply `designation` instead of `semester`/`gender`/`distance_km`.

**Login** uses `application/x-www-form-urlencoded` (standard OAuth2 password flow), not JSON:
```
username=stu00001&password=student123
```

## Student — `/api/student` (role: student only)

| Method | Path | Description |
|---|---|---|
| GET | `/api/student/profile` | The logged-in student's profile. |
| GET | `/api/student/dashboard` | Full dashboard payload: profile, overall + subject-wise attendance, monthly trend, leave/marks summary, latest prediction. |
| GET | `/api/student/attendance` | Full attendance history + monthly trend. |

All three operate only on the currently authenticated student — there is no way to pass another student's code through these routes.

---

## Faculty — `/api/faculty` (role: faculty or admin)

| Method | Path | Query Params | Description |
|---|---|---|---|
| GET | `/api/faculty/dashboard` | `department` (optional) | Institution/department-wide stats, comparisons, defaulters, improving/declining lists, risk distribution. |
| GET | `/api/faculty/students` | `department`, `semester`, `search`, `skip`, `limit` | Paginated, filterable, searchable student list. |
| GET | `/api/faculty/students/{student_code}` | — | Full dashboard-style drill-down into one student. |
| GET | `/api/faculty/below-75` | `department` (optional) | All students currently below 75% attendance. |
| GET | `/api/faculty/top-defaulters` | `limit` (default 10), `department` (optional) | Lowest-attendance students. |

## Prediction — `/api/prediction` (any authenticated role)

| Method | Path | Description |
|---|---|---|
| POST | `/api/prediction/predict` | Run all 3 models (risk, forecast, segmentation) + generate AI recommendations. |
| GET | `/api/prediction/history/{student_code}` | Past predictions for a student, newest first. |
| GET | `/api/prediction/model-info` | Training metrics for all 3 models (accuracy/F1/MAE/R²/silhouette). |

**Predict request body** — either supply `student_code` (pulls live features from the DB) **or** all six manual fields:
```json
{ "student_code": "STU00001" }
```
```json
{
  "current_attendance": 68,
  "previous_attendance": 74,
  "leaves_taken": 11,
  "internal_marks": 45,
  "distance_km": 18.5,
  "monthly_attendance": 65
}
```
Access rule: students may only predict for themselves (omit `student_code` or pass their own); faculty/admin may predict for any student or run a manual what-if scenario (`student_code` omitted).

**Response:**
```json
{
  "student_code": "STU00001",
  "predicted_risk_level": "High Risk",
  "risk_model_used": "RandomForest",
  "risk_confidence": 0.94,
  "forecast_attendance_percentage": 64.2,
  "forecast_model_used": "RandomForestRegressor",
  "cluster_segment": "Critical",
  "ai_recommendations": [
    "Your attendance is below the 75% requirement. Attend your next 5 classes continuously to recover.",
    "You are at high risk of detention. Please speak with your faculty advisor as soon as possible."
  ],
  "created_at": "2026-07-14T10:32:00"
}
```

## Export — `/api/export`

| Method | Path | Auth | Query Params | Description |
|---|---|---|---|---|
| GET | `/api/export/student-report` | Any | `format` (`pdf`\|`csv`), `student_code` (required for faculty, omitted for a student's own report) | Downloads a formatted attendance report. |
| GET | `/api/export/faculty-report` | Faculty/admin | `format`, `department` (optional) | Downloads a class/department summary report. |

Both return a file stream with `Content-Disposition: attachment`.

---

## Error Response Shape

All errors follow FastAPI's standard shape:
```json
{ "detail": "Human-readable error message" }
```
Common status codes: `400` (bad request/validation), `401` (missing/invalid/expired token), `403` (role/ownership violation), `404` (not found), `422` (request body validation failure), `503` (a required ML/analytics artifact hasn't been generated yet — run the corresponding training/EDA script).
